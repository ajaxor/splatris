# Splatris

Splatris is a barebones mobile-friendly browser game starter that can be hosted for free with GitHub Pages.

## Run it locally

Because this app is plain HTML, CSS, and JavaScript, you can open `index.html` directly in a browser. For a local web server that behaves more like GitHub Pages, run:

```bash
python3 -m http.server 8080
```

Then open <http://localhost:8080>.

## Publish it on GitHub Pages

1. Push this repository to GitHub.
2. In GitHub, open the repository settings.
3. Go to **Pages**.
4. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
5. Choose your default branch and the repository root folder, then save.
6. After GitHub finishes publishing, open the Pages URL shown on that settings page.

The published link usually looks like `https://YOUR-USERNAME.github.io/splatris/`.

## Multiplayer demo

The page includes a no-build WebRTC data-channel demo for up to 4 players on the same local network:

1. Serve the folder from the host device, for example `python3 -m http.server 8080`.
2. Open the host page from the host's LAN address, such as `http://192.168.1.25:8080`.
3. Tap **Host room**. The host creates one QR code for each guest player slot.
4. Each guest scans a different QR code. Their phone will show a WebRTC answer string.
5. Copy or send that answer back to the host, paste it into **Paste guest answer on host**, and tap **Connect guest**.
6. Once connected, each phone can toggle its own player button and the state propagates to every connected device.

This is intentionally a simple peer-to-peer demonstration. The QR code carries the host's WebRTC offer, and the answer is pasted back manually so the demo can run from static files without a signaling server. The QR image is generated by `api.qrserver.com`; if you need a fully offline LAN demo, replace that image URL with a local QR-code generator.
